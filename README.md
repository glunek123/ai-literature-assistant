# AI文献助手

基于RAG和大语言模型的智能文献检索与总结系统，面向大学生快速了解研究领域核心观点。

## 项目简介

本系统通过构建向量知识库，实现语义检索相关文献，并使用大语言模型自动总结核心观点，帮助大学生快速了解研究领域的核心内容。

### 核心功能

- **知识库构建**：从Excel数据构建向量知识库
- **语义检索**：基于语义相似度检索相关文献
- **智能总结**：使用大语言模型总结文献核心观点
- **Web界面**：友好的用户交互界面

## 技术架构

### 核心技术栈

| 技术 | 用途 | 选择理由 |
|------|------|----------|
| LangChain | RAG应用框架 | 完整的工具链，开发效率高 |
| ChromaDB | 向量数据库 | 轻量级，无需额外安装，适合Demo |
| DashScope API | Embedding和大模型 | 中文效果好，API调用简单 |
| Streamlit | Web框架 | 专为数据科学设计，开发速度快 |
| Pandas | 数据处理 | Python数据分析标准库 |

### 系统架构

```
用户输入主题
    ↓
RAG知识库检索（语义匹配）
    ↓
按相关性排序，取Top 3篇
    ↓
大模型总结核心观点
    ↓
结构化输出：文献列表 + 核心观点总结
```

## 安装步骤

### 1. 环境要求

- Python 3.8+
- pip包管理器

### 2. 安装依赖

```bash
pip install -r requirements.txt
```

### 3. 配置API密钥

在 `src/config.py` 文件中配置你的DashScope API密钥：

```python
DASHSCOPE_API_KEY = "your-api-key-here"
```

### 4. 准备数据

将论文数据放在 `data/CNKI_1_1.xlsx` 文件中，包含以下列：
- Title-题名
- Author-作者
- Organ-单位
- Source-文献来源
- Keyword-关键词
- Summary-摘要
- PubTime-发表时间

## 使用方法

### 1. 构建知识库

首次使用需要构建知识库：

```bash
python build_kb.py
```

### 2. 启动Web应用

```bash
streamlit run app.py
```

### 3. 使用Web界面

1. 在浏览器中打开 `http://localhost:8501`
2. 输入你感兴趣的研究主题
3. 点击"开始检索"按钮
4. 查看检索到的文献和核心观点总结

## 防止幻觉策略

1. **严格的Prompt设计**：明确要求基于提供的文献内容总结
2. **完整的上下文**：将检索到的文献完整信息传递给大模型
3. **输出验证**：检查输出格式是否符合要求

## 性能优化建议

1. **向量数据库优化**：使用更高效的向量数据库如Milvus或Pinecone
2. **缓存机制**：对常见查询结果进行缓存
3. **批量处理**：支持批量查询和总结
4. **异步处理**：使用异步IO提升响应速度

## 常见问题

### Q: 知识库构建失败怎么办？
A: 检查Excel文件路径是否正确，确保数据格式符合要求。

### Q: 检索结果不准确怎么办？
A: 尝试使用更具体的关键词，或者调整TOP_K参数增加检索数量。

### Q: 大模型总结质量不高怎么办？
A: 检查Prompt模板，确保提供足够的上下文信息。
