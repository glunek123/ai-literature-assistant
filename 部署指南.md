## 🚂 Railway 部署（推荐）

### 为什么选择 Railway？
- ✅ **完全免费**：每月提供 $5 的免费额度
- ✅ **自动识别 Dockerfile**：无需复杂配置
- ✅ **支持环境变量**：安全存储 API 密钥
- ✅ **自动部署**：代码更新后自动重新部署
- ✅ **公网访问**：自动生成 HTTPS 域名

### 部署步骤

#### 第一步：注册 Railway 账号
1. 访问 [Railway](https://railway.app/)
2. 使用 GitHub 账号登录

#### 第二步：创建新项目
1. 点击 **"New Project"**
2. 选择 **"Deploy from GitHub repo"**
3. 选择你的仓库：`ai-literature-assistant`

#### 第三步：配置环境变量
1. 在项目页面，点击 **"Variables"**
2. 添加以下环境变量：
   ```
   DASHSCOPE_API_KEY=sk-2033afec579e49998b4289759508b739
   ```

#### 第四步：部署
1. Railway 会自动识别 Dockerfile 并开始构建
2. 等待部署完成（约 2-5 分钟）
3. 部署成功后，点击 **"Generate Domain"** 获取公网 URL

#### 第五步：访问应用
- 点击生成的域名即可访问应用
- 格式：`https://your-app.up.railway.app`

### 故障排查
- **构建失败**：检查日志，确保 Dockerfile 配置正确
- **应用无法启动**：检查环境变量是否设置正确
- **内存不足**：Railway 免费版有内存限制，考虑升级到付费版

## 🔺 Vercel 部署

### 部署步骤

#### 第一步：注册 Vercel 账号
1. 访问 [Vercel](https://vercel.com/)
2. 使用 GitHub 账号登录

#### 第二步：导入项目
1. 点击 **"Add New Project"**
2. 选择 **"Import Git Repository"**
3. 选择你的仓库：`ai-literature-assistant`

#### 第三步：配置环境变量
1. 在配置页面，点击 **"Environment Variables"**
2. 添加环境变量：
   - Name: `DASHSCOPE_API_KEY`
   - Value: `sk-2033afec579e49998b4289759508b739`
   - Environment: 选择所有环境（Production、Preview、Development）

#### 第四步：部署
1. 点击 **"Deploy"**
2. 等待部署完成

### 注意事项
⚠️ Vercel 对 Streamlit 的支持有限，可能需要额外配置。如果遇到问题，建议使用 Railway。

## ☁️ 阿里云 ECS 部署

### 准备工作
- 购买阿里云 ECS 实例（推荐配置：2 核 2G，约 50 元/月）
- 系统：Ubuntu 20.04 或 CentOS 7

### 部署步骤

#### 第一步：连接服务器
```bash
ssh root@你的服务器 IP
```

#### 第二步：安装 Docker
```bash
# 更新系统
apt update && apt upgrade -y

# 安装 Docker
curl -fsSL https://get.docker.com | sh

# 启动 Docker
systemctl start docker
systemctl enable docker
```

#### 第三步：克隆项目
```bash
git clone https://github.com/glunek123/ai-literature-assistant.git
cd ai-literature-assistant
```

#### 第四步：构建 Docker 镜像
```bash
docker build -t ai-literature-assistant .
```

#### 第五步：运行容器
```bash
docker run -d \
  --name ai-literature-assistant \
  -p 8501:8501 \
  -e DASHSCOPE_API_KEY=sk-2033afec579e49998b4289759508b739 \
  --restart always \
  ai-literature-assistant
```

#### 第六步：配置安全组
1. 登录阿里云控制台
2. 进入 ECS 实例 → 安全组
3. 添加规则：开放 8501 端口

#### 第七步：访问应用
```
http://你的服务器 IP:8501
```

### 常用命令
```bash
# 查看日志
docker logs -f ai-literature-assistant

# 重启容器
docker restart ai-literature-assistant

# 停止容器
docker stop ai-literature-assistant

# 删除容器
docker rm -f ai-literature-assistant
```

## 🆘 常见问题

### Q1: Railway 部署后无法访问？
A: 检查以下几点：
1. 确认容器已启动（查看 Railway 日志）
2. 确认环境变量已设置
3. 确认端口配置正确（8501）

### Q2: 部署后知识库在哪里？
A: Railway 和 Vercel 是临时文件系统，重启后数据会丢失。建议：
1. 使用外部数据库（如 Chroma Cloud）
2. 或者每次重启后重新构建知识库

### Q3: 如何更新部署？
A: 只需推送代码到 GitHub，平台会自动重新部署。

### Q4: API 密钥安全吗？
A: 使用环境变量存储 API 密钥是安全的做法，不要将密钥硬编码在代码中。
